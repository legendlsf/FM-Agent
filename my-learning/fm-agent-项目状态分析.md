## 六、当前限制和约束

### 6.1 技术限制

1. **仅支持 OpenRouter API**
   - 不支持直接调用 OpenAI/Anthropic API
   - 原因：需要高并发和灵活的速率限制

2. **模型依赖性强**
   - 强烈依赖 Claude Opus/Sonnet 4.6+
   - 弱模型可能产生不可靠结果

3. **语言支持有限**
   - 仅支持 10 种主流语言
   - 不支持 Kotlin, Swift, Scala 等

4. **代码规模限制**
   - 单个函数不宜超过 1000 行
   - 超长函数会被分块处理

### 6.2 使用约束

1. **API 成本**
   - 大型项目分析成本较高
   - 10K 行代码约消耗 $5-20（取决于模型）

2. **时间成本**
   - 10K 行代码约需 30-60 分钟
   - 100K 行代码可能需要数小时

3. **准确性限制**
   - 发现的 Bug 需要人工审查
   - 可能存在误报（false positive）
   - 可能遗漏 Bug（false negative）

## 八、与其他工具对比

### 8.1 静态分析工具

| 工具 | 类型 | 优势 | 劣势 |
|------|------|------|------|
| FM-Agent | LLM 驱动形式化 | 深度推理，发现语义 Bug | 成本高，速度慢 |
| SonarQube | 静态分析 | 快速，成本低 | 浅层检查，误报多 |
| Coverity | 静态分析 | 工业级，误报少 | 无法推理复杂逻辑 |
| Infer | 静态分析 | 开源，Facebook 出品 | 语言支持有限 |

### 8.2 形式化验证工具

| 工具 | 类型 | 优势 | 劣势 |
|------|------|------|------|
| FM-Agent | 自动化 | 无需专家，自动生成规范 | 依赖 LLM 质量 |
| Coq | 定理证明 | 数学严格，100% 正确 | 学习曲线陡峭 |
| TLA+ | 模型检测 | 适合并发系统 | 需要手写规范 |
| Dafny | 验证语言 | 自动验证 | 需要重写代码 |

### 8.3 测试工具

| 工具 | 类型 | 优势 | 劣势 |
|------|------|------|------|
| FM-Agent | 规范驱动 | 自动生成测试用例 | 覆盖率不保证 |
| AFL | 模糊测试 | 发现崩溃 Bug | 无法推理逻辑错误 |
| QuickCheck | 属性测试 | 随机生成测试 | 需要手写属性 |
| Hypothesis | 属性测试 | Python 生态好 | 需要手写假设 |
